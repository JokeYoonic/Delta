## 未成功：Logto 管理后台

### 问题
访问 http://localhost:3302 创建 SPA/M2M 应用时，@logto/react SDK 自动请求 `urn:logto:resource:organizations` 资源，
但 Logto OSS v1.36.0 的 organizations 功能未完整配置，导致 token 交换返回 403，React 组件崩溃。

### 错误堆栈 (多次复现)
```
at http://localhost:3302/console/assets/@logto-EY-wsPGP.js:1:16194
at async pe (@logto-EY-wsPGP.js:1:3061)
at async et.getAccessTokenByRefreshToken (@logto-EY-wsPGP.js:1:13711)
at async @logto-EY-wsPGP.js:1:9970 (x2)
at async et.getOrganizationTokenClaims (@logto-EY-wsPGP.js:1:11114)
at async react-SGHKQULX.js:43:26868
at async index-BmZ5-Clk.js:84:393628
```

### 尝试过的修复 (均无效，原因是浏览器缓存或补丁不完整)
1. 手动创建 admin 用户 → 注册 UI 不显示登录入口 (sign_in_mode=Register)
2. 改 sign_in_mode=SignInAndRegister → 登录成功但 admin console 崩溃
3. 创建 organization resource + scope + 角色授权 → 403 依旧
4. Patch `getOrganizationTokenClaims` (try-catch) → 浏览器缓存未加载
5. Patch `getAccessTokenByRefreshToken` (try-catch) → 浏览器缓存未加载
6. 清空所有 org scope/resource 引用 → 浏览器缓存未加载

### 关键发现
- Logto OSS 容器内无 python3，node v22.22.2 可用
- Admin console 源码在 `/etc/logto/packages/console/src/App.tsx`
- @logto/react SDK 在 `/etc/logto/node_modules/.pnpm/@logto+react@4.0.13/`
- @logto/client 源码 `#getOrganizationToken` 在 client.js:359
- M2M 应用的 secret 存在 DB 但经过哈希，无法直接用于 API 调用
- Admin 用户已通过 UI 成功注册 (username: admin)

### 根本原因
Logto v1.36.0 OSS 的 @logto/react SDK 默认请求 organizations 资源，
但服务器端 organizations 功能 (urn:logto:resource:organizations) 未被正确处理，
refresh_token grant 对该资源返回 403。

### 可能的解决路径 (未尝试)
1. 升级 Logto 到更新的版本 (修复了这个 bug)
2. 用 Logto Cloud 代替自部署
3. 用其他认证方案 (Auth0, Clerk, Supabase Auth 等)
4. 手动修改 Logto 源码 `packages/console/src/App.tsx`，移除 UserScope.Organizations 后重新构建
